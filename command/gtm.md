# COMMAND — GTM Playbook
Last updated: 260413

## Fratricide Pain Story (founder-sourced)

### The Story
Two Claude Code sessions running simultaneously on the same project. One building a feature, the other doing a targeted fix. Neither knew the other existed. The first committed everything to git before the second could selectively stage its changes — one session's work overwrote the other's intent. Classic fratricide.

### The Pain
Both agents were competent. Both were doing good work. But they were environmentally blind to each other — operating as if they were the only actor in the workspace. Operator became the damage control relay, untangling commits that should never have collided.

### The Line (use verbatim in calls)
> "Two AI agents working on the same project committed each other's work without knowing the other existed. I spent the next hour untangling it. That's the moment I knew — if I'm the only thing preventing my agents from stepping on each other, I'm not the boss. I'm the traffic cop."

### When to deploy
- After Q4 (Visibility Question): "Right now, if I asked you what your AI agents are doing at this exact moment — could you tell me?"
- When prospect describes tool chaos or parallel AI usage
- When prospect laughs about "hoping it's running" — escalate: "It gets worse. Let me tell you what happened to me last week."

### What it validates
- **RELAY FAILURE** — operator as human middleware between own tools
- **VISIBILITY FAILURE** — no unified view of what agents are touching
- **Product thesis:** COMMAND is air traffic control. When you route through it, it's the conductor. When you don't, you get fratricide.
